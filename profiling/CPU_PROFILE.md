# CPU profile — SIMD effectiveness & optimization log

_Generated by `python profiling/analyze.py` from criterion medians (`target/criterion`). Re-run `cargo bench -p neuralforge_core` to refresh._

## `dot_product` — scalar vs AVX2 + FMA

| Dim | Scalar (ns) | AVX2+FMA (ns) | Speedup | SIMD throughput |
|----:|------------:|--------------:|:-------:|----------------:|
| 128 | 31.3 | 4.06 | **7.7×** | 31.5 Gelem/s |
| 384 | 124.5 | 10.27 | **12.1×** | 37.4 Gelem/s |
| 768 | 270.0 | 23.31 | **11.6×** | 32.9 Gelem/s |
| 1536 | 567.3 | 45.77 | **12.4×** | 33.6 Gelem/s |

## Reading the numbers

- **Vectorization win.** The AVX2+FMA kernel processes 8 `f32` lanes per
  instruction and carries four independent accumulators to hide the ~4-cycle
  FMA latency, then does one horizontal reduction. The ~7× → ~12× speedup as
  dimension grows is the loop/reduction overhead being amortized.
- **Throughput ceiling.** From 384-dim on, the kernel saturates around
  **~33 Gelem/s** — close to the limit for unaligned `f32` loads on one core
  (Core Ultra 9, AVX2; AVX-512 is fused off). Past this point the kernel is
  load-bound, not compute-bound: more ILP would not help.
- **Where to parallelize.** Because a single similarity is bandwidth-bound,
  the win at corpus scale comes from `rayon` fan-out across vectors, not from
  a wider single-vector kernel — which is exactly how `batch_similarity` and
  `top_k_search` are built.

## Optimization log

| Hypothesis | Change | Measured effect |
|------------|--------|-----------------|
| Scalar dot is compute-bound on FP throughput | Hand-vectorize with AVX2+FMA, runtime `is_x86_feature_detected!` dispatch | **7–12× faster** (table above) |
| One FMA accumulator stalls on latency | Four independent accumulators (ILP) | Saturates the ~33 Gelem/s load ceiling from 384-dim |
| Top-k must not materialize all scores | Per-thread bounded min-heaps merged at end | `O(k·threads)` memory vs `O(n)`; see PERFORMANCE.md §3 |
| Batch normalization recomputed per query | Precompute corpus norms once | `O(n·d)` instead of `O(q·n·d)` normalization |
